Critical Orphan spinner process on Ctrl+C — terminal corruption install.sh:10,80-95
The start_spinner() function spawns a while true background process and disowns it. The top-level EXIT trap only prints "Installer interrupted" — it does NOT call stop_spinner(). When a user hits Ctrl+C during a spinner, the main script exits but the spinner process keeps running forever, printing escape codes to the terminal. The user sees garbled output and must kill %1 or close the terminal. 
trap 'echo -e "\n${RED}✗ Installer interrupted..."' EXIT  # ← never kills spinner
start_spinner() {
  ...
  done &
  spin_pid=$!
  disown "$spin_pid"  # ← now unkillable by parent trap
Fix: Replace the EXIT trap with a cleanup function that kills spin_pid, restores cursor, then prints the interrupt message.
Critical .env file created with world-readable permissions (0644) install.sh:setup_api_keys()
API keys (ANTHROPIC_API_KEY, AWS credentials, etc.) are written to ~/.pi/agent/.env with default umask permissions — typically 0644. Any user on the system can read it. On shared dev servers or multi-user Macs, this leaks credentials. 
cp "$env_template" "$env_file"   # ← inherits default perms (644)
# No chmod 600 anywhere
Fix: Add chmod 600 "$env_file" immediately after creating/copying the .env file. Also add to verify.sh as a check.
